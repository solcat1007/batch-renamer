# BatchRenamer — 批量重命名

> 为剪辑素材文件夹中的文件进行批量重命名，支持自定义模板（前缀 + 序号 + 后缀）、实时预览和一键撤销。

## 功能列表

- **灵活命名模板**：自定义前缀、起始序号、序号位数（前导零）、步长、后缀
- **实时预览**：修改模板参数即时显示新文件名效果，无需实际执行
- **重名冲突检测**：执行前自动检查新文件名是否重复或与现有文件冲突，冲突时弹窗提示
- **安全重命名策略**：两步临时名策略（先全部改临时名 → 再改为目标名），避免中间冲突
- **一键撤销**：记录最近一次重命名操作，可完整还原为原始文件名
- **错误回滚**：重命名过程中出错时自动尝试回滚已执行步骤
- **纯标准库**：零外部依赖，仅使用 Python 标准库（tkinter / os / json / pathlib）


## 详细使用

### 启动

```bash
python batch_renamer.py
```

### 操作步骤

1. 点击 **「选择文件夹」**，选择包含待重命名文件的目录
2. 设置命名模板参数：
   - **前缀**：放在序号前面的文字（如 `素材_`）
   - **起始序号**：第一个文件的编号（如 `1`）
   - **位数**：序号的数字位数，不足补零（如 `3` 则显示 `001`）
   - **步长**：序号递增间隔（如 `1` 则 001, 002, 003；`5` 则 001, 006, 011）
   - **后缀**：放在序号后面的文字（如 `_航拍`）
3. 文件列表中实时显示每条记录 `原文件名 → 新文件名`
4. 确认无误后点击 **「执行重命名」**
5. 如需还原，点击 **「撤销重命名」**

### 命名模板示例

| 模板设置 | 效果示例 |
|---|---|
| 前缀=`素材_` 起始=`1` 位数=`3` 步长=`1` 后缀=`` | `IMG_0001.MOV` → `素材_001.MOV` |
| 前缀=`Shot` 起始=`10` 位数=`2` 步长=`10` 后缀=`_final` | `DSC_0023.mp4` → `Shot10_final.mp4` … `Shot20_final.mp4` |
| 前缀=`` 起始=`0` 位数=`4` 步长=`1` 后缀=`` | `随机名.aac` → `0000.aac` … `0005.aac` |

### 命令行参数

本项目为图形化（GUI）工具，不提供命令行参数。

## 输入输出说明

### 输入

- 选择文件夹后，自动列出目录下所有常规文件（不含隐藏文件、不含子目录、不含 `.` 开头的系统文件）
- 文件按名称排序排列

### 输出

- 撤销数据存储在内存中，程序关闭后无法撤销之前的操作

## 常见问题

**Q: 重命名后发现搞错了，能恢复吗？**
A: 如果未关闭程序，点击「撤销重命名」即可一键还原。一旦关闭程序，撤销数据丢失，无法通过工具还原。建议重命名前仔细核对预览效果。

**Q: 新文件名与已有文件冲突怎么办？**
A: 执行前会自动检测冲突并弹出确认对话框，列出冲突文件名。你可以选择继续（覆盖）或取消后调整模板。

**Q: 可以仅重命名特定类型的文件吗？**
A: 当前版本不支持按类型过滤，会重命名文件夹内的所有文件。如需只处理视频文件，请先将其他文件移出该文件夹。

**Q: 序号位数设为 3，但文件数量超过 999 怎么办？**
A: 位数只决定前导零补足，不会截断序号。例如 `起始=1 位数=3`，第 1500 个文件将命名为 `1500`（4 位数）。

**Q: 支持子文件夹递归处理吗？**
A: 当前版本仅处理所选文件夹的直接文件，不递归子目录。
